# 01 项目总览与路线图

> 状态：教程蓝图
>
> 适用范围：帮助你快速建立项目全貌，并区分“当前仓库已有能力”和“完整项目目标能力”。

---

## 1. 项目在解决什么问题

一句话版：

> 用户上传自己的资料，系统先从资料里检索相关内容，再基于检索结果回答问题，而不是脱离知识库胡乱作答。

这个项目的价值不只是“做一个聊天接口”，而是把下面几件事串起来：

- 用户体系
- 知识库管理
- 文档管理
- 文档切片与检索
- 问答会话持久化
- 面向真实产品的接口组织

---

## 2. 当前仓库与最终目标

### 2.1 当前仓库已实现

- 用户注册、登录、刷新令牌、退出登录、获取当前用户
- 知识库增删改查
- MongoDB 接入
- JWT 鉴权
- Swagger 文档
- 统一响应结构

### 2.2 完整目标蓝图

- 一个用户有多个知识库
- 一个知识库有多个文档
- 一个文档有多个 chunk
- 一个知识库有多个问答会话
- 一个问答会话有多条消息
- 文档列表支持按文件类型区分，例如 `word`、`pdf`、`markdown`
- 支持基于知识库的 RAG 问答与 SSE 流式输出

---

## 3. 核心名词表

| 名词 | 说明 |
| --- | --- |
| 用户 | 系统登录主体 |
| 知识库 | 一组文档的容器，例如“前端面试资料库” |
| 文档 | 一份上传的文件或导入的文本 |
| Chunk | 文档切分后的最小检索单元 |
| 会话 | 某个知识库内的一次连续问答上下文 |
| 消息 | 会话中的一条输入或输出 |
| RAG | 先检索、再生成回答 |
| Embedding | 把文本转换成向量表示 |
| SSE | 服务端流式返回文本片段 |

---

## 4. 技术选型

### 4.1 当前代码已采用

- 后端框架：`NestJS 11`
- 数据库：`MongoDB + Mongoose`
- 鉴权：`JWT accessToken + refreshToken`
- 文档：`Swagger`
- 包管理器：`pnpm`

### 4.2 完整项目预计采用

- 文档存储：对象存储，例如 `Cloudflare R2`
- 向量检索：`MongoDB Atlas Vector Search`
- 模型接入：OpenAI 兼容聊天模型与 Embedding 模型
- AI 编排：`LangChain`

这里要特别注意：

- 上面这些 AI 相关能力是教程蓝图的一部分，不代表当前仓库已经具备
- 当前部署文档只会要求你准备当前代码真正需要的变量和服务

---

## 6. 当前阶段建议

如果你正在基于当前仓库继续往后做，优先顺序建议是：

1. `documents`
2. `document_chunks`
3. `chat_sessions` / `chat_messages`
4. RAG 检索
5. SSE

这个顺序的好处是：

- 前端文档列表能最早得到支持
- 数据模型的依赖关系自然展开
- 后续 AI 能力能建立在稳定的文档主链路上
