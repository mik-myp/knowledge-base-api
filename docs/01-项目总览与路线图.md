# 项目总览与路线图

## 1. 当前最小路线

当前路线只有一条：

1. `users`
2. `knowledge_bases`
3. `documents`
4. `document_chunks`
5. `chat_sessions`
6. `chat_messages`
7. `chat/ask`

不要再并行引入：

- `chat/stream`

这些都不是当前个人 RAG 最小闭环必需项。

---

## 2. 当前代码与文档的分工

### 已在代码中存在

- 登录鉴权
- 知识库管理
- 文档上传与编辑器录入
- 文档归属校验

### 只在文档中定义，等待你自己实现

- chunk schema
- 会话 schema
- 消息 schema
- 文档拆分服务
- 问答接口

---

## 3. 当前阶段的完成标准

满足下面几点，就算个人 RAG 最小闭环完成：

1. 用户能登录
2. 用户能创建知识库
3. 用户能上传或录入文档
4. 文档能生成 chunk
5. 用户能创建或继续一个会话
6. 用户能对知识库提问并得到基于 chunk 的回答
7. 用户消息和 AI 消息能被保存
