# 项目产品与业务总览

## 1. 目标闭环

当前项目只围绕一个个人使用的 RAG 最小闭环：

1. 用户登录
2. 创建知识库
3. 上传文档或录入 Markdown 文本
4. 文档被拆分为 chunk
5. 创建或继续一个问答会话
6. 基于 chunk 做 AI 问答并保存消息

这就是本轮设计与文档的唯一目标。

---

## 2. 只保留最小闭环需要的能力

当前不再展开这些内容：

- 流式输出
- 向量索引优化
- 各类统计缓存字段
- 各类处理流水线状态字段

如果一个字段不直接服务于上面的 5 步，就不放进当前最小方案。

---

## 3. 当前仓库中的已存在部分

当前代码里已经存在并已被收口优化的部分：

- `users`
- `knowledge_bases`
- `documents`
- `storage`

这些模块已经被调整为更适合个人 RAG 最小闭环的基础版本。

---

## 4. 当前仓库中仍缺失的部分

为了满足“不要直接替你实现不存在的部分”，当前缺失能力只记录在 docs：

- `document_chunks`
- `chat_sessions`
- `chat_messages`
- 文档拆分逻辑
- `POST /chat/ask`

你可以直接按文档继续实现。
