# 最小闭环实施步骤

## 1. 先跑通已有部分

先验证当前仓库已存在的能力：

1. 注册或登录
2. 创建知识库
3. 创建编辑器文档
4. 查询文档详情，确认 `content` 已落库

---

## 2. 再补你要自己实现的部分

在当前代码基础上，你只需要继续补两段：

1. `documents -> document_chunks`
2. `document_chunks -> chat_sessions/chat_messages -> chat/ask`

---

## 3. 文档拆分实现顺序

推荐顺序：

1. 先支持编辑器 Markdown 文本拆分
2. 再支持 `md` 上传文件拆分
3. 再支持 `pdf`
4. 最后支持 `word`

这样最容易先把问答跑通。

---

## 4. 问答实现顺序

推荐只做非流式：

1. 接收 `knowledgeBaseId + question + 可选 sessionId`
2. 创建或继续 `chat_session`
3. 先保存一条 `human` 消息
4. 检索当前知识库 chunk
5. 把命中 chunk 组装进 prompt
6. 调用 LangChain chat model
7. 保存一条 `ai` 消息和 `sources`
8. 返回 `sessionId + answer + sources`

当前阶段不要先做流式。
