# knowledge-base-api 项目部署与更新指南

> 适用项目：`knowledge-base-api`
> 
> 适用场景：阿里云云服务器、`Ubuntu 24.04`、`Nest + MongoDB` 同机部署、`PM2 + Nginx`
> 
> 文档目的：记录当前项目已经验证通过的一套部署、更新、排错流程，后续可以直接按本文执行

---

## 1. 当前项目关键信息

基于当前仓库代码，项目有这些特征：

- 包管理器：`pnpm`
- 框架：`NestJS 11`
- 数据库：`MongoDB`
- 进程管理：`PM2`
- 反向代理：`Nginx`
- 默认端口：`3000`
- Swagger 地址：`/docs`
- 无全局 `/api` 前缀

当前常用接口：

- `POST /users/register`
- `POST /users/login`
- `POST /users/refresh`
- `POST /users/logout`
- `GET /users/me`
- `GET /knowledge-bases`
- `POST /knowledge-bases`
- `GET /knowledge-bases/:id`
- `PATCH /knowledge-bases/:id`
- `DELETE /knowledge-bases/:id`

## 2. 本文记录的部署方案

本项目当前采用的是这套方案：

- 云服务器系统：`Ubuntu 24.04`
- Node.js：`22`
- MongoDB：`8.0`
- 项目目录：`/srv/knowledge-base-api`
- PM2 进程名：`knowledge-base-api`
- MongoDB 与 Nest 项目部署在同一台服务器
- 对外只开放 `80/443`，应用和数据库只监听本机

这套方案适合：

- 个人学习
- 练手项目
- 低并发测试环境

这套方案不适合：

- 高并发生产环境
- 高可用部署
- 多机分布式场景

## 3. 首次部署的完整流程

### 3.1 服务器安全组

阿里云安全组只放行：

- `22`
- `80`
- `443`

不要放行：

- `3000`
- `27017`

### 3.2 安装系统基础软件

```bash
apt update && apt upgrade -y
apt install -y curl git ca-certificates gnupg lsb-release build-essential nginx ufw jq
```

### 3.3 配置 swap

`2核2G` 机器建议开启 `2G swap`：

```bash
fallocate -l 2G /swapfile
chmod 600 /swapfile
mkswap /swapfile
swapon /swapfile
grep -q '^/swapfile ' /etc/fstab || echo '/swapfile none swap sw 0 0' >> /etc/fstab
echo 'vm.swappiness=1' > /etc/sysctl.d/99-swappiness.conf
sysctl --system
free -h
```

### 3.4 配置防火墙

```bash
ufw allow OpenSSH
ufw allow 'Nginx Full'
ufw --force enable
ufw status
```

### 3.5 安装 Node.js、pnpm、PM2

```bash
curl -fsSL https://deb.nodesource.com/setup_22.x | bash -
apt install -y nodejs
node -v
npm -v

npm install -g pnpm pm2
pnpm -v
pm2 -v
```

### 3.6 安装 MongoDB

```bash
curl -fsSL https://www.mongodb.org/static/pgp/server-8.0.asc | gpg -o /usr/share/keyrings/mongodb-server-8.0.gpg --dearmor
echo "deb [ arch=amd64,arm64 signed-by=/usr/share/keyrings/mongodb-server-8.0.gpg ] https://repo.mongodb.org/apt/ubuntu noble/mongodb-org/8.0 multiverse" > /etc/apt/sources.list.d/mongodb-org-8.0.list
apt update
apt install -y mongodb-org
systemctl enable mongod
systemctl start mongod
systemctl status mongod --no-pager
```

### 3.7 创建 MongoDB 业务库用户

```bash
export MONGO_DB=knowledge-base-api
export MONGO_USER=kbapi
export MONGO_PASS=KbapiPass2026Strong
```

```bash
mongosh <<EOF
use $MONGO_DB
db.createUser({
  user: "$MONGO_USER",
  pwd: "$MONGO_PASS",
  roles: [{ role: "readWrite", db: "$MONGO_DB" }]
})
EOF
```

### 3.8 开启 MongoDB 身份认证

```bash
cp /etc/mongod.conf /etc/mongod.conf.bak
sed -i 's/^  bindIp: .*/  bindIp: 127.0.0.1/' /etc/mongod.conf
grep -q '^security:' /etc/mongod.conf || printf '\nsecurity:\n  authorization: enabled\n' >> /etc/mongod.conf
systemctl restart mongod
systemctl status mongod --no-pager
```

### 3.9 测试 MongoDB 连接

```bash
mongosh "mongodb://$MONGO_USER:$MONGO_PASS@127.0.0.1:27017/$MONGO_DB?authSource=$MONGO_DB" --eval "db.runCommand({ ping: 1 })"
```

如果返回里有 `ok: 1`，说明数据库连接正常。

### 3.10 拉取项目代码

```bash
mkdir -p /srv
git clone https://github.com/mik-myp/knowledge-base-api.git /srv/knowledge-base-api
cd /srv/knowledge-base-api
```

### 3.11 创建项目环境变量

示例：

```bash
cat > /srv/knowledge-base-api/.env <<EOF
NODE_ENV=production
PORT=3000
MONGODB_URI=mongodb://kbapi:KbapiPass2026Strong@127.0.0.1:27017/knowledge-base-api?authSource=knowledge-base-api
JWT_ACCESS_SECRET=knowledge-base-api-access-please-change-this-to-a-long-random-string
JWT_ACCESS_EXPIRESIN=15m
JWT_REFRESH_SECRET=knowledge-base-api-refresh-please-change-this-to-a-long-random-string
JWT_REFRESH_EXPIRESIN=30d
EOF
```

建议再执行：

```bash
chmod 600 /srv/knowledge-base-api/.env
```

### 3.12 安装依赖并构建

```bash
cd /srv/knowledge-base-api
pnpm install --frozen-lockfile
pnpm build
```

### 3.13 手动启动测试

```bash
cd /srv/knowledge-base-api
pnpm start:prod
```

新开一个终端测试：

```bash
curl http://127.0.0.1:3000/
curl http://127.0.0.1:3000/docs
```

如果没问题，再按 `Ctrl + C` 停掉当前前台进程。

### 3.14 使用 PM2 托管

```bash
cd /srv/knowledge-base-api
pm2 start dist/main.js --name knowledge-base-api --cwd /srv/knowledge-base-api
pm2 save
pm2 startup systemd -u root --hp /root
pm2 status
pm2 logs knowledge-base-api --lines 100
```

### 3.15 配置 Nginx

```bash
cat > /etc/nginx/sites-available/knowledge-base-api <<EOF
server {
    listen 80;
    server_name _;

    client_max_body_size 20m;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;

        proxy_set_header Host \$host;
        proxy_set_header X-Real-IP \$remote_addr;
        proxy_set_header X-Forwarded-For \$proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto \$scheme;
    }
}
EOF
```

启用配置：

```bash
ln -sf /etc/nginx/sites-available/knowledge-base-api /etc/nginx/sites-enabled/knowledge-base-api
rm -f /etc/nginx/sites-enabled/default
nginx -t
systemctl restart nginx
systemctl enable nginx
```

### 3.16 首次部署完成后的访问地址

- 首页：`http://服务器公网IP/`
- Swagger：`http://服务器公网IP/docs`

## 4. 当前项目实际使用的环境变量

按照当前代码，生产环境最核心的是这几项：

```env
NODE_ENV=production
PORT=3000
MONGODB_URI=mongodb://kbapi:KbapiPass2026Strong@127.0.0.1:27017/knowledge-base-api?authSource=knowledge-base-api
JWT_ACCESS_SECRET=replace-with-a-long-random-string
JWT_ACCESS_EXPIRESIN=15m
JWT_REFRESH_SECRET=replace-with-another-long-random-string
JWT_REFRESH_EXPIRESIN=30d
```

说明：

- `PORT`：Nest 服务监听端口
- `MONGODB_URI`：MongoDB 连接串
- `JWT_ACCESS_SECRET`：accessToken 签名密钥，生产环境一定要改
- `JWT_ACCESS_EXPIRESIN`：accessToken 有效期
- `JWT_REFRESH_SECRET`：refreshToken 签名密钥，生产环境一定要改
- `JWT_REFRESH_EXPIRESIN`：refreshToken 有效期

## 5. 项目更新代码的标准流程

如果本地代码已经提交并推送到了远程仓库，服务器更新就执行下面这套：

```bash
cd /srv/knowledge-base-api
git pull
pnpm install --frozen-lockfile
pnpm build
pm2 restart knowledge-base-api
pm2 logs knowledge-base-api --lines 50
```

### 5.1 每一步的作用

- `git pull`
  拉取远程最新代码
- `pnpm install --frozen-lockfile`
  保证依赖与锁文件一致
- `pnpm build`
  重新生成 `dist`
- `pm2 restart knowledge-base-api`
  重启服务加载新代码
- `pm2 logs ...`
  查看更新后是否报错

### 5.2 如果只是改了业务代码，没有改依赖

可以简化成：

```bash
cd /srv/knowledge-base-api
git pull
pnpm build
pm2 restart knowledge-base-api
```

### 5.3 如果改了依赖

只要这些文件发生变化：

- `package.json`
- `pnpm-lock.yaml`

就必须执行：

```bash
pnpm install --frozen-lockfile
```

### 5.4 如果改了环境变量

修改 `.env` 后，至少要重启服务：

```bash
cd /srv/knowledge-base-api
pm2 restart knowledge-base-api
```

## 6. 推荐的更新脚本

为了避免每次手动敲多条命令，可以在服务器项目目录里放一个脚本。

创建文件：

```bash
cat > /srv/knowledge-base-api/deploy.sh <<'EOF'
#!/usr/bin/env bash
set -e

cd /srv/knowledge-base-api
git pull
pnpm install --frozen-lockfile
pnpm build
pm2 restart knowledge-base-api
pm2 logs knowledge-base-api --lines 50
EOF
```

赋予执行权限：

```bash
chmod +x /srv/knowledge-base-api/deploy.sh
```

以后更新：

```bash
cd /srv/knowledge-base-api
./deploy.sh
```

## 7. 接口自测命令

### 7.1 注册

```bash
curl -s -X POST http://127.0.0.1:3000/users/register \
  -H "Content-Type: application/json" \
  -d '{
    "email": "admin@example.com",
    "password": "12345678",
    "username": "admin"
  }'
```

### 7.2 登录

```bash
curl -s -X POST http://127.0.0.1:3000/users/login \
  -H "Content-Type: application/json" \
  -d '{
    "email": "admin@example.com",
    "password": "12345678"
  }'
```

### 7.3 获取 accessToken

```bash
LOGIN_RESPONSE=$(curl -s -X POST http://127.0.0.1:3000/users/login \
  -H "Content-Type: application/json" \
  -d '{
    "email": "admin@example.com",
    "password": "12345678"
  }')

ACCESS_TOKEN=$(echo "$LOGIN_RESPONSE" | jq -r '.data.accessToken')
REFRESH_TOKEN=$(echo "$LOGIN_RESPONSE" | jq -r '.data.refreshToken')
echo "$ACCESS_TOKEN"
echo "$REFRESH_TOKEN"
```

### 7.4 获取当前用户

```bash
curl -s http://127.0.0.1:3000/users/me \
  -H "Authorization: Bearer $ACCESS_TOKEN" | jq
```

### 7.5 创建知识库

```bash
CREATE_KB_RESPONSE=$(curl -s -X POST http://127.0.0.1:3000/knowledge-bases \
  -H "Authorization: Bearer $ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "前端面试知识库",
    "description": "个人练习环境创建的第一条知识库"
  }')

echo "$CREATE_KB_RESPONSE" | jq
KB_ID=$(echo "$CREATE_KB_RESPONSE" | jq -r '.data.id')
echo "$KB_ID"
```

### 7.6 查询知识库列表

```bash
curl -s "http://127.0.0.1:3000/knowledge-bases?page=1&pageSize=10" \
  -H "Authorization: Bearer $ACCESS_TOKEN" | jq
```

### 7.7 查询单个知识库

```bash
curl -s "http://127.0.0.1:3000/knowledge-bases/$KB_ID" \
  -H "Authorization: Bearer $ACCESS_TOKEN" | jq
```

### 7.8 更新知识库

```bash
curl -s -X PATCH "http://127.0.0.1:3000/knowledge-bases/$KB_ID" \
  -H "Authorization: Bearer $ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "更新后的知识库名称",
    "description": "更新后的描述"
  }' | jq
```

### 7.9 刷新 Token

```bash
curl -s -X POST http://127.0.0.1:3000/users/refresh \
  -H "Content-Type: application/json" \
  -d "{
    \"refreshToken\": \"$REFRESH_TOKEN\"
  }" | jq
```

### 7.10 退出登录

```bash
curl -s -X POST http://127.0.0.1:3000/users/logout \
  -H "Authorization: Bearer $ACCESS_TOKEN" | jq
```

### 7.11 删除知识库

```bash
curl -s -X DELETE "http://127.0.0.1:3000/knowledge-bases/$KB_ID" \
  -H "Authorization: Bearer $ACCESS_TOKEN" | jq
```

## 9. 常见问题排查

### 9.1 外网打不开页面

优先检查：

- 阿里云安全组有没有开 `80`
- `nginx -t` 是否通过
- `pm2 status` 里项目是否在线

### 9.2 Swagger 打不开

检查：

```bash
curl http://127.0.0.1:3000/docs
curl http://127.0.0.1/docs
```

如果本机能通，公网不通，优先查 Nginx 和安全组。

### 9.3 登录报 Token 问题

重点检查：

- `.env` 里 `JWT_ACCESS_SECRET` 和 `JWT_REFRESH_SECRET` 是否存在
- 重启后是否确实加载了新的 `.env`
- 是否执行了 `pm2 restart knowledge-base-api`

### 9.4 项目连不上 MongoDB

重点检查：

- `MONGODB_URI` 是否写对
- `authSource` 是否正确
- `mongod` 是否正常运行
- MongoDB 用户是否创建成功

### 9.5 更新代码后启动失败

依次检查：

```bash
cd /srv/knowledge-base-api
git pull
pnpm install --frozen-lockfile
pnpm build
pm2 restart knowledge-base-api
pm2 logs knowledge-base-api --lines 200
```

如果是依赖问题，通常是漏执行了 `pnpm install --frozen-lockfile`。

## 10. 日常使用建议

- 不要直接对外暴露 `3000` 和 `27017`
- 每次更新代码，优先走固定流程，不要跳步骤
- 只要改了依赖，就重新执行 `pnpm install --frozen-lockfile`
- 只要改了环境变量，就重启 `PM2`
- 在正式做更大改动前，先跑一遍最小接口自测
