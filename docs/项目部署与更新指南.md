# knowledge-base-api 项目部署与更新指南

> 状态：当前已实现
>
> 适用项目：`knowledge-base-api`
>
> 适用场景：当前仓库真实可部署状态，基于 `NestJS + MongoDB + pnpm + PM2 + Nginx`
>
> 适用日期：2026-03-19

---

## 1. 先明确当前部署范围

这份文档只覆盖当前仓库已经真正实现的后端能力：

- 用户注册、登录、刷新、退出、当前用户信息
- 知识库增删改查
- Swagger 文档
- MongoDB 持久化

当前部署不要求这些能力已经接入：

- 文档上传
- `GET /documents`
- RAG 问答
- SSE
- Cloudflare R2
- MongoDB Atlas Vector Search
- OpenAI 兼容模型

上面这些属于后续扩展，不是当前部署前提。

## 2. 当前项目关键信息

- 包管理器：`pnpm`
- 框架：`NestJS 11`
- 数据库：`MongoDB`
- 进程管理：`PM2`
- 反向代理：`Nginx`
- 默认端口：`3000`
- Swagger 地址：`/docs`
- 无全局 `/api` 前缀

当前真实接口：

- `POST /users/register`
- `POST /users/login`
- `POST /users/refresh`
- `POST /users/logout`
- `GET /users/me`
- `GET /knowledge-bases`
- `POST /knowledge-bases`
- `GET /knowledge-bases/:id`
- `PATCH /knowledge-bases/:id`
- `DELETE /knowledge-bases/:id`

这些接口的返回结构说明，可配合 [03-当前已实现接口与代码结构](./03-当前已实现接口与代码结构.md) 一起看。

## 3. 推荐部署方案

本项目当前采用的推荐方案：

- 服务器系统：`Ubuntu 24.04`
- Node.js：`22`
- MongoDB：`8.0`
- 项目目录：`/srv/knowledge-base-api`
- PM2 进程名：`knowledge-base-api`
- MongoDB 与 Nest 服务同机部署
- 对外只开放 `80/443`

适用：

- 个人练习
- 测试环境
- 低并发后端服务

不适用：

- 高可用生产集群
- 多机分布式部署
- 大规模并发写入

## 4. 首次部署流程

## 4.1 安装基础软件

```bash
apt update && apt upgrade -y
apt install -y curl git ca-certificates gnupg lsb-release build-essential nginx ufw jq
```

## 4.2 安装 Node.js、pnpm、PM2

```bash
curl -fsSL https://deb.nodesource.com/setup_22.x | bash -
apt install -y nodejs
npm install -g pnpm pm2

node -v
pnpm -v
pm2 -v
```

## 4.3 安装 MongoDB

```bash
curl -fsSL https://www.mongodb.org/static/pgp/server-8.0.asc | gpg -o /usr/share/keyrings/mongodb-server-8.0.gpg --dearmor
echo "deb [ arch=amd64,arm64 signed-by=/usr/share/keyrings/mongodb-server-8.0.gpg ] https://repo.mongodb.org/apt/ubuntu noble/mongodb-org/8.0 multiverse" > /etc/apt/sources.list.d/mongodb-org-8.0.list
apt update
apt install -y mongodb-org
systemctl enable mongod
systemctl start mongod
systemctl status mongod --no-pager
```

## 4.4 配置防火墙

```bash
ufw allow OpenSSH
ufw allow 'Nginx Full'
ufw --force enable
ufw status
```

## 4.5 拉取项目代码

```bash
mkdir -p /srv
git clone https://github.com/mik-myp/knowledge-base-api.git /srv/knowledge-base-api
cd /srv/knowledge-base-api
```

## 4.6 创建环境变量

当前代码真正需要的最小环境变量如下：

```env
NODE_ENV=production
PORT=3000
MONGODB_URI=mongodb://kbapi:KbapiPass2026Strong@127.0.0.1:27017/knowledge-base-api?authSource=knowledge-base-api
JWT_ACCESS_SECRET=replace-with-a-long-random-string
JWT_ACCESS_EXPIRESIN=15m
JWT_REFRESH_SECRET=replace-with-another-long-random-string
JWT_REFRESH_EXPIRESIN=30d
```

建议：

```bash
chmod 600 /srv/knowledge-base-api/.env
```

说明：

- `MONGODB_URI` 是当前唯一不可缺少的核心变量
- JWT 相关变量在代码里存在默认兜底，但生产环境必须显式配置
- `R2 / 向量检索 / OpenAI` 相关变量不属于当前部署前提

## 4.7 安装依赖并构建

```bash
cd /srv/knowledge-base-api
pnpm install --frozen-lockfile
pnpm build
```

## 4.8 手动启动验证

```bash
cd /srv/knowledge-base-api
pnpm start:prod
```

新开一个终端验证：

```bash
curl http://127.0.0.1:3000/
curl http://127.0.0.1:3000/docs
```

## 4.9 使用 PM2 托管

```bash
cd /srv/knowledge-base-api
pm2 start dist/main.js --name knowledge-base-api --cwd /srv/knowledge-base-api
pm2 save
pm2 startup systemd -u root --hp /root
pm2 status
```

## 4.10 配置 Nginx

```bash
cat > /etc/nginx/sites-available/knowledge-base-api <<EOF
server {
    listen 80;
    server_name _;

    client_max_body_size 20m;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;

        proxy_set_header Host \$host;
        proxy_set_header X-Real-IP \$remote_addr;
        proxy_set_header X-Forwarded-For \$proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto \$scheme;
    }
}
EOF
```

启用配置：

```bash
ln -sf /etc/nginx/sites-available/knowledge-base-api /etc/nginx/sites-enabled/knowledge-base-api
rm -f /etc/nginx/sites-enabled/default
nginx -t
systemctl restart nginx
systemctl enable nginx
```

## 6. 标准更新流程

如果代码已经提交到远程仓库，服务器更新按下面流程执行：

```bash
cd /srv/knowledge-base-api
git pull
pnpm install --frozen-lockfile
pnpm build
pm2 restart knowledge-base-api
pm2 logs knowledge-base-api --lines 50
```

### 6.1 什么时候可以省略安装依赖

如果这次更新没有改这些文件：

- `package.json`
- `pnpm-lock.yaml`

可以简化为：

```bash
cd /srv/knowledge-base-api
git pull
pnpm build
pm2 restart knowledge-base-api
```

### 6.2 如果改了环境变量

修改 `.env` 后至少执行：

```bash
cd /srv/knowledge-base-api
pm2 restart knowledge-base-api
```

## 8. 接口自测命令

当前所有外部接口最终都会返回统一结构：

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {},
  "timestamp": "2026-03-19T00:00:00.000Z",
  "path": "/users/login"
}
```

说明：

- `code`、`message`、`timestamp`、`path` 来自统一响应封装
- 业务主体数据统一放在 `data`
- 列表接口重点看 `data.dataList` 和 `data.total`
- 单条详情接口重点看 `data.id`、`data.name`、`data.documentCount`、`data.chunkCount`、`data.sessionCount`

## 8.1 注册

```bash
curl -s -X POST http://127.0.0.1:3000/users/register \
  -H "Content-Type: application/json" \
  -d '{
    "email": "admin@example.com",
    "password": "12345678",
    "username": "admin"
  }' | jq
```

## 8.2 登录并提取令牌

```bash
LOGIN_RESPONSE=$(curl -s -X POST http://127.0.0.1:3000/users/login \
  -H "Content-Type: application/json" \
  -d '{
    "email": "admin@example.com",
    "password": "12345678"
  }')

ACCESS_TOKEN=$(echo "$LOGIN_RESPONSE" | jq -r '.data.accessToken // .data.data.accessToken')
REFRESH_TOKEN=$(echo "$LOGIN_RESPONSE" | jq -r '.data.refreshToken // .data.data.refreshToken')
echo "$ACCESS_TOKEN"
echo "$REFRESH_TOKEN"
```

## 8.3 获取当前用户

```bash
curl -s http://127.0.0.1:3000/users/me \
  -H "Authorization: Bearer $ACCESS_TOKEN" | jq
```

## 8.4 创建知识库

```bash
CREATE_KB_RESPONSE=$(curl -s -X POST http://127.0.0.1:3000/knowledge-bases \
  -H "Authorization: Bearer $ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "前端面试知识库",
    "description": "当前仓库阶段的示例知识库"
  }')

echo "$CREATE_KB_RESPONSE" | jq
KB_ID=$(echo "$CREATE_KB_RESPONSE" | jq -r '.data.id // .data.data.id')
echo "$KB_ID"
```

## 8.5 查询知识库列表

```bash
curl -s "http://127.0.0.1:3000/knowledge-bases?page=1&pageSize=10" \
  -H "Authorization: Bearer $ACCESS_TOKEN" | jq
```

预期关键结构：

- `data.dataList`
- `data.total`

## 8.6 查询单个知识库

```bash
curl -s "http://127.0.0.1:3000/knowledge-bases/$KB_ID" \
  -H "Authorization: Bearer $ACCESS_TOKEN" | jq
```

预期关键字段：

- `data.name`
- `data.documentCount`
- `data.chunkCount`
- `data.sessionCount`

## 8.7 更新知识库

```bash
curl -s -X PATCH "http://127.0.0.1:3000/knowledge-bases/$KB_ID" \
  -H "Authorization: Bearer $ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "更新后的知识库名称",
    "description": "更新后的描述"
  }' | jq
```

## 8.8 刷新 Token

```bash
curl -s -X POST http://127.0.0.1:3000/users/refresh \
  -H "Content-Type: application/json" \
  -d "{
    \"refreshToken\": \"$REFRESH_TOKEN\"
  }" | jq
```

## 8.9 退出登录

```bash
curl -s -X POST http://127.0.0.1:3000/users/logout \
  -H "Authorization: Bearer $ACCESS_TOKEN" | jq
```

## 8.10 删除知识库

```bash
curl -s -X DELETE "http://127.0.0.1:3000/knowledge-bases/$KB_ID" \
  -H "Authorization: Bearer $ACCESS_TOKEN" | jq
```
