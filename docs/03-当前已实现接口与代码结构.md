## 3. 当前已经完成的代码优化

### ID 处理统一

- 接口层用 `string`
- 数据库层用 `ObjectId`

### 知识库字段收口

知识库只保留：

- `userId`
- `name`
- `description`

### 文档字段收口

文档只保留当前最小闭环需要的主记录字段。

### 编辑器正文落库

- `POST /documents/editor` 已经会把 Markdown 原文保存到 `documents.content`

### Markdown 上传正文落库

- `md` / `text` 上传内容会同步写入 `documents.content`

### 对象存储不再是最小闭环硬前置

- 如果对象存储未配置，当前只允许上传 `md` / `txt`
- `pdf` / `doc` / `docx` 需要对象存储可用后再上传
- 这样可以避免生成既没有原文件也没有正文的无效文档记录

### 上传类型限制

- 当前已有代码只允许上传 `md`、`txt`、`pdf`、`doc`、`docx`
