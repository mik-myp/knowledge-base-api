## 2. 当前可用接口

## 2.1 用户接口

| 方法 | 路径 | 说明 | 鉴权 |
| --- | --- | --- | --- |
| `POST` | `/users/register` | 用户注册 | 否 |
| `POST` | `/users/login` | 用户登录 | 否 |
| `POST` | `/users/refresh` | 刷新令牌 | 否 |
| `POST` | `/users/logout` | 退出登录 | 是 |
| `GET` | `/users/me` | 获取当前用户 | 是 |

## 2.2 知识库接口

| 方法 | 路径 | 说明 | 鉴权 |
| --- | --- | --- | --- |
| `POST` | `/knowledge-bases` | 创建知识库主记录 | 是 |
| `GET` | `/knowledge-bases` | 获取知识库列表 | 是 |
| `GET` | `/knowledge-bases/:id` | 获取知识库详情 | 是 |
| `PATCH` | `/knowledge-bases/:id` | 更新知识库 | 是 |
| `DELETE` | `/knowledge-bases/:id` | 删除知识库 | 是 |

当前没有这些接口：

- `GET /documents`
- `POST /documents/upload`
- `POST /chat/ask`
- `POST /chat/stream`

这些都属于后续阶段能力。

## 3. 当前响应结构

当前项目启用了全局响应拦截器，最终 HTTP 响应会统一为：

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {},
  "timestamp": "2026-03-19T00:00:00.000Z",
  "path": "/knowledge-bases"
}
```

其中：

- `users` 控制器会先通过 `ResponseUtil` 返回标准结构
- `knowledge-bases` 控制器返回业务数据后，再由全局拦截器包一层标准结构

对外表现是一致的。

## 4.3 当前知识库记录结构

当前知识库主记录包含这些关键字段：

- `id`
- `userId`
- `name`
- `description`
- `documentCount`
- `chunkCount`
- `sessionCount`
- `lastIndexedAt`

这里的 `documentCount`、`chunkCount`、`sessionCount` 是汇总字段。

## 6. 当前 Swagger 边界

当前 Swagger 应该只展示两类真实接口：

- `users`
- `knowledge-bases`

当前知识库接口的语义边界是：

- 只管理知识库主数据
- 只返回知识库汇总字段
- 不承诺已经实现文档、会话、消息等子资源
