# 代码级任务清单与实施顺序

## 0. 先说明这份文档怎么读

这份文档是写给下面这类开发者的：

1. 初学 NestJS
2. 初学 LangChain
3. 初学 MongoDB / Mongoose

所以整份文档都按同一个结构来写，每个阶段固定回答 5 个问题：

1. 这个阶段要做什么
2. 这个阶段要改哪些文件
3. 这个阶段要创建哪些方法，方法写在哪个文件
4. 这些方法分别是做什么的
5. 最小可用代码怎么写

你可以把这份文档当成“施工手册”，不要跳着看。

推荐阅读顺序：

1. 先看 `1. 总体顺序`
2. 再从 `阶段 0` 开始往下做
3. 前一个阶段没做完，不要做后一个阶段

---

## 1. 总体顺序

当前项目建议按下面顺序实现：

1. 阶段 0：准备依赖、环境变量、MongoDB Atlas Vector Search 索引
2. 阶段 1：补 `document_chunks`
3. 阶段 2：补 `DocumentIndexingService`
4. 阶段 3：把索引流程接入 `documents`
5. 阶段 4：补 `chat_sessions` 和 `chat_messages`
6. 阶段 5：实现 `POST /chat/ask`
7. 阶段 6：最后再考虑 RAG Agent

为什么是这个顺序？

因为 `chat/ask` 依赖前面所有东西：

```text
documents
  -> document_chunks
  -> document_chunk_vectors
      -> semantic search
          -> chat_sessions
              -> chat_messages
                  -> POST /chat/ask
```

## 2. 阶段 0：准备基础设施

### 2.1 这个阶段要做什么

这个阶段不直接写业务功能，先把后面会用到的基础能力准备好。

你要完成 4 件事：

1. 安装 LangChain 和向量检索依赖
2. 准备环境变量
3. 确认 embedding 维度
4. 在 MongoDB Atlas 创建 Vector Search Index

### 2.2 这个阶段要改哪些文件

这个阶段主要改配置，不改业务模块。

需要处理的地方：

1. `package.json`
2. `.env.example`
3. `.env.local`
4. MongoDB Atlas 后台配置

### 2.3 这个阶段要创建哪些方法

这个阶段暂时不在 `src/` 下创建正式业务方法。

只需要临时准备一个最小脚本，用来确认 embedding 维度。

### 2.4 第一步：安装依赖

先执行：

```bash
pnpm add @langchain/community @langchain/textsplitters @langchain/mongodb mongodb pdf-parse mammoth
```

这些依赖分别负责：

1. `@langchain/community`：PDF loader
2. `@langchain/textsplitters`：文本切分
3. `@langchain/mongodb`：MongoDB Atlas 向量存储
4. `mongodb`：原生 Mongo Collection 类型
5. `pdf-parse`：PDF 解析底层依赖
6. `mammoth`：`docx` 解析

### 2.5 第二步：补环境变量

你当前已经有：

1. `MONGODB_URI`
2. `OPENAI_CHAT_MODEL`
3. `OPENAI_CHAT_API_KEY`
4. `OPENAI_CHAT_BASE_URL`
5. `OPENAI_EMBEDDING_MODEL`
6. `OPENAI_EMBEDDING_API_KEY`
7. `OPENAI_EMBEDDING_BASE_URL`

现在再补两个：

```env
MONGODB_VECTOR_COLLECTION=document_chunk_vectors
MONGODB_VECTOR_INDEX=document_chunk_vector_index
```

说明：

1. `MONGODB_VECTOR_COLLECTION`：向量集合名
2. `MONGODB_VECTOR_INDEX`：Atlas 向量索引名

### 2.6 第三步：确认 embedding 维度

MongoDB Atlas Vector Search 建索引时，必须知道 embedding 的维度。

不要猜，不要照抄别人项目，必须自己实际跑出来。

#### 临时脚本示例

你可以先写一个临时脚本，核心逻辑如下：

```ts
import { OpenAIEmbeddings } from '@langchain/openai';

async function main() {
  const embeddings = new OpenAIEmbeddings({
    model: process.env.OPENAI_EMBEDDING_MODEL,
    apiKey: process.env.OPENAI_EMBEDDING_API_KEY,
    configuration: {
      baseURL: process.env.OPENAI_EMBEDDING_BASE_URL,
    },
  });

  const vector = await embeddings.embedQuery('hello world');
  console.log(vector.length);
}

void main();
```

你真正要看的是：

1. `console.log(vector.length)`

这个数字就是你后面要填进 Atlas 索引里的 `numDimensions`。

### 2.7 第四步：创建 Atlas Vector Search Index

在 `document_chunk_vectors` 集合上创建 Atlas Vector Search Index。

建议至少包含下面这些字段：

1. `embedding`：向量字段
2. `userId`：过滤字段
3. `knowledgeBaseId`：过滤字段
4. `documentId`：过滤字段

为什么还要给 `userId` 和 `knowledgeBaseId` 建 filter？

因为当前项目是多用户、多知识库的。

你后面检索时必须做到：

1. 只搜当前用户的数据
2. 只搜当前知识库的数据

#### Atlas 索引示例

下面是一个可参考模板，唯一要改的是 `numDimensions`：

```json
{
  "fields": [
    {
      "type": "vector",
      "path": "embedding",
      "numDimensions": 1536,
      "similarity": "cosine"
    },
    {
      "type": "filter",
      "path": "userId"
    },
    {
      "type": "filter",
      "path": "knowledgeBaseId"
    },
    {
      "type": "filter",
      "path": "documentId"
    }
  ]
}
```

### 2.8 这个阶段完成后你要检查什么

全部做完后，你要确认：

1. 依赖已安装成功
2. embedding 维度已经实际打印过
3. Atlas Vector Index 已创建成功
4. `.env.local` 里已经有向量集合名和索引名

## 3. 阶段 1：补 `document_chunks`

### 3.1 这个阶段要做什么

这个阶段只做一件大事：

1. 把“文档切分后的业务 chunk”落到 MongoDB 里

注意，这里还不做向量检索。

你现在只需要先把业务表补出来。

### 3.2 这个阶段要改哪些文件

需要新增：

1. `src/documents/schemas/document_chunk.schema.ts`

需要修改：

1. `src/documents/documents.module.ts`
2. `src/documents/documents.service.ts`

### 3.3 这个阶段要创建哪些方法

都写在：

1. `src/documents/documents.service.ts`

需要创建：

1. `replaceDocumentChunks`
2. `deleteDocumentChunks`

这两个方法的职责分别是：

1. `replaceDocumentChunks`：删除旧 chunk，再写入新 chunk
2. `deleteDocumentChunks`：删除某个文档的全部 chunk

### 3.4 第一步：创建 `document_chunk` schema

文件：

1. `src/documents/schemas/document_chunk.schema.ts`

最小可用代码：

```ts
import { Prop, Schema, SchemaFactory } from '@nestjs/mongoose';
import { HydratedDocument, Types } from 'mongoose';

export type DocumentChunkDocument = HydratedDocument<DocumentChunk>;

@Schema({
  collection: 'document_chunks',
  timestamps: true,
  versionKey: false,
})
export class DocumentChunk {
  @Prop({ type: Types.ObjectId, ref: 'User', required: true, index: true })
  userId: Types.ObjectId;

  @Prop({
    type: Types.ObjectId,
    ref: 'KnowledgeBase',
    required: true,
    index: true,
  })
  knowledgeBaseId: Types.ObjectId;

  @Prop({ type: Types.ObjectId, ref: 'Document', required: true, index: true })
  documentId: Types.ObjectId;

  @Prop({ type: Number, required: true, min: 0 })
  sequence: number;

  @Prop({ type: String, required: true })
  content: string;

  @Prop({ type: Number, min: 1, default: undefined })
  page?: number;

  @Prop({ type: Number, min: 0, default: undefined })
  startIndex?: number;

  @Prop({ type: Number, min: 0, default: undefined })
  endIndex?: number;
}

export const DocumentChunkSchema =
  SchemaFactory.createForClass(DocumentChunk);

DocumentChunkSchema.index({ documentId: 1, sequence: 1 }, { unique: true });
DocumentChunkSchema.index({ userId: 1, knowledgeBaseId: 1 });
DocumentChunkSchema.index({ knowledgeBaseId: 1, documentId: 1 });
```

### 3.7 第四步：创建 `replaceDocumentChunks`

文件：

1. `src/documents/documents.service.ts`

这个方法要做什么：

1. 先把这个文档旧的 chunk 全删掉
2. 如果新 chunk 是空数组，直接结束
3. 否则按顺序批量写入

最小可用代码：

```ts
private async replaceDocumentChunks(
  userId: string,
  knowledgeBaseId: string,
  documentId: string,
  chunks: Array<{
    content: string;
    page?: number;
    startIndex?: number;
    endIndex?: number;
  }>,
) {
  const userObjectId = toObjectId(userId);
  const knowledgeBaseObjectId = toObjectId(knowledgeBaseId);
  const documentObjectId = toObjectId(documentId);

  await this.documentChunkModel
    .deleteMany({
      userId: userObjectId,
      documentId: documentObjectId,
    })
    .exec();

  if (!chunks.length) {
    return;
  }

  const payload = chunks.map((chunk, index) => ({
    userId: userObjectId,
    knowledgeBaseId: knowledgeBaseObjectId,
    documentId: documentObjectId,
    sequence: index,
    content: chunk.content,
    page: chunk.page,
    startIndex: chunk.startIndex,
    endIndex: chunk.endIndex,
  }));

  await this.documentChunkModel.insertMany(payload);
}
```

### 3.8 第五步：创建 `deleteDocumentChunks`

文件：

1. `src/documents/documents.service.ts`

这个方法要做什么：

1. 删除某个文档对应的全部 chunk

最小可用代码：

```ts
private async deleteDocumentChunks(userId: string, documentId: string) {
  await this.documentChunkModel
    .deleteMany({
      userId: toObjectId(userId),
      documentId: toObjectId(documentId),
    })
    .exec();
}
```

### 3.9 第六步：在 `remove()` 里接入 `deleteDocumentChunks`

文件：

1. `src/documents/documents.service.ts`

你只需要在删除文档时补一行：

```ts
await this.deleteDocumentChunks(userId, id);
```

建议顺序：

1. 先找到文档
2. 如果有对象存储文件就删文件
3. 删除 chunk
4. 删除主文档

### 3.10 这个阶段完成后你要检查什么

做完后请确认：

1. `document_chunks` 集合已经能生成
2. `DocumentsService` 里已经有这两个方法
3. 删除文档时会联动删除 chunk
